# Plan: Fix Unsplash API Images for Recipes

## Root Cause

The Unsplash API is not returning images because **the `UNSPLASH_ACCESS_KEY` secret is not configured** in the production environment.

Evidence:

- `src/routes/images.ts:38-43`: API returns 503 "Unsplash API key not configured" when key is missing
- `wrangler.toml:16`: Comments indicate the secret should be set via `wrangler secret put`
- `src/types.ts:31`: UNSPLASH_ACCESS_KEY is optional (`?`), confirming it may not be set
- No `.dev.vars` file exists for local development

## Why Images Don't Load

When the frontend calls `/api/images/recipe/:query`:

1. The API checks `env.UNSPLASH_ACCESS_KEY`
2. If undefined/empty, returns 503 with error message
3. Frontend gets no image (silent failure - line 1113 catches and logs but doesn't display)

## Fix Steps

1. **Add UNSPLASH_ACCESS_KEY to `.dev.vars`** for local development
   - Create `.dev.vars` file with: `UNSPLASH_ACCESS_KEY=your_key_here`
   - Already in `.gitignore` (line 44), safe to create

2. **Set production secret** via command line:

   ```
   wrangler secret put UNSPLASH_ACCESS_KEY
   ```

   - Get key from: https://unsplash.com/developers
   - Note: Free tier includes 50 requests/hour

3. **Debug**: If key is set but still failing:
   - Run locally with `npm run dev`
   - Check browser DevTools console for `/api/images` responses
   - Check if Unsplash API returning errors

4. **Verification**: Test with a known recipe and confirm images load.

## Note on API Response

The current code silently fails when images don't load (see `src/index.ts:1113-1115`). Consider improving error visibility in the UI for better debugging.
